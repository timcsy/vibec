# Vibec 專案憲法

## 核心原則

### I. 前端靜態網站優先

本專案採用前端靜態網站架構。所有核心功能應能編譯為純靜態資產（HTML、CSS、JavaScript），不依賴伺服器端渲染或動態後端邏輯。網站應能完全部署至 GitHub Pages 或任何靜態主機服務。

### II. GitHub Pages 相容性

所有內容、資源路徑和連結必須相容 GitHub Pages 部署環境。使用相對路徑替代絕對路徑；避免依賴伺服器端功能（如重寫規則、API 路由）；資產應通過 CDN 或相對路徑載入。

### III. 無伺服器架構

不應引入伺服器端運行時依賴。如需動態功能，使用無伺服器方案（如 GitHub Actions、Netlify Functions）或第三方 API。禁止添加 Node.js 服務器、Python 後端或其他需要持續運行的伺服器。

### IV. 無障礙性和包容設計

所有網頁內容必須符合 WCAG 2.1 AA 標準。完整的語義化 HTML、正確的顏色對比度、鍵盤導航支援、螢幕閱讀器相容性為必需。每個新功能需進行無障礙性測試。

### V. 不要過度設計

只實現當前需要的功能。不預先設計未來可能用不到的架構或功能。遵循 YAGNI 原則（You Aren't Gonna Need It）。保持程式碼簡單易懂。

### VI. 工具謹慎使用

允許使用 Vite 等建構工具。但使用工具時必須小心，不得覆蓋或丟失原本的程式碼和資產。所有工具配置必須備份原始文件。

## 治理

本憲法優先於其他實踐規範。所有 PR 和審查必須驗證是否符合上述原則。修訂需文件記錄、批准，並在本檔案更新。
